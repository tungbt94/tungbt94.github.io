# Xử lý lấy 1 field trong object lồng nhau có thể bị null trong JS

Cho cấu trúc 1 object:

User {
  id: string,
  name: string,
  addresses: [{
    street: string,
    suburb: string,
    postcode: string,
    country: string,
    mail: [{
      subject: string,
      sentOn: string,
      body: string,
    }]
  }]
}

Ta có thể lấy body của email đầu tiên của địa chỉ thứ nhất bằng cách user.addresses[0].mail[0].body được không?

Cách này sẽ gây lỗi nếu addresses hoặc addresses[0] hoặc addresses[0].mail bị null hoặc undefined. Vậy có những cách giải quyết nào?

# Cách 1: basic nhất

const firstEmailBody =
  user.addresses &&
  user.addresses[0] &&
  user.addresses[0].mail &&
  user.addresses[0].mail[0] &&
  user.addresses[0].mail[0].body

Hoặc có thể viết ultility function

const deepGet = (obj, keys) => keys.reduce((xs, x) => (xs && xs[x] ? xs[x] : null), obj);
const firstEmailBody = deepGet(user, ['user', 'addresses', '0', 'mail', '0', 'body'])

# Cách 2: dùng idx (opens new window) của facebook

const firstEmailBody = idx(user, u => user.addresses[0].mail[0].body)

# Cách 3: dùng Maybe của true-myth (opens new window)

const firstEmailBody = Maybe.fromNullable(user.addresses)
  .map(addresses => addresses[0])
  .map(firstAddress => firstAddress.mail)
  .map(allMail => allMail[0])
  .map(firstSentMail => firstSentMail.body)
  .unwrapOr("never");

# Cách 4: dùng Optional chaining

Đây là feature rất hay từ TC39. Tuy nhiên hiện tại chưa được hỗ trợ native nên cần dùng babel.

const firstEmailBody = user.addresses?.[0]?.mail?.[0]?.body
